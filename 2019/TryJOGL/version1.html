Version 1：まず中身のないウィンドウ（フレーム）を1枚開く

さっそく「tryjogl」ディレクトリ（パッケージ）にある「Example.java」の中身を、以下のソースコードと同じになるように直してください。編集するところをハイライトで表しておきます。「Version 0」を「Version 1」にしましょう。オーソドックスなウィンドウの開き方を会得できると思います。

ソースコード（Version 1）

package tryjogl;

import javax.swing.JFrame;

/**
 * JOGL（Java bindings for OpenGL）の例題プログラムである。
 * OpenGLを使ったJavaによる三次元グラフィックスの初歩的な練習になる。
 */
public class Example extends Object
{
    /**
     * Exampleのコンストラクタである。<br>
     */
    public Example()
    {
        System.out.println("Example");

        JFrame aFrame = new JFrame("Wire Teapot");
        aFrame.addNotify();
        int titleBarHeight = aFrame.getInsets().top;
        aFrame.setSize(400, 400 + titleBarHeight);
        aFrame.setLocationRelativeTo(null);
        aFrame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        aFrame.setFocusable(false);
        aFrame.setVisible(true);

        return;
    }

    /**
     * 針金細工のティーポット（Wire Teapot）を描くOpenGLのウィンドウを開く。
     * @param arguments コマンドの引数列（文字列の配列）
     */
    public static void main(String[] arguments)
    {
        System.out.println("main");

        new Example();

        return;
    }
}

プログラムの解説

javax.swing.JFrameのインスタンスを生成し、その際に「Wire Teapot」をタイトルに指定します。ウィンドウの中のGUIコンポーネントのサイズの決定（計算）は、フレームのインスタンスに「addNotify()」のメッセージを送信したときになります。その後に「getInsets()」のメッセージを発信すれば、フレームのタイトル・バーの高さがわかります。したがって、開くべきウィンドウの大きさを正確に求めることができます。その様子は下図のようになります。

ウィンドウの大きさを算出した後、そのウィンドウを開く場所を指定するのに「setLocationRelativeTo(Component)」のメッセージを用います。引数にnullを与えることにより、スクリーン（ディスプレイ）の中央に来るようにウィンドウの場所が計算されます。

次に、ウィンドウを閉じる際の処理を「setDefaultCloseOperation(int)」で指定します。引数にJFrame.EXIT_ON_CLOSEを与えることにより、System.exit(0)すなわちアプリケーションの終了が行われることになります。

今後（Version 2以降）、ウィンドウの中身（コンポーネント）としてOpenGLのキャンバスを入れて、キーボードやマウスのフォーカスを得ることになりますゆえ、フレームがフォーカスを必要とすることはないでしょう。それを指示しておくために「setFocusable(boolean)」のメッセージを用い、引数にfalseを与えて伝達しておきます。

以上でウィンドウを開く準備が整いますので、引数にtrueを与えた「setVisible(boolean)」のメッセージを発信して、実際にウィンドウを開くようにプログラミングしました。
